# 剧本创作 Skill

## 触发条件
用户输入 `/screenplay` 或要求"写剧本"、"创作剧本"、"剧本创作"时激活本 skill。

---

## 第一步：理解创作意图（必须先问，再写）

在动笔之前，先向用户确认以下信息（可以一次性提问，不要逐一追问）：

### 必问项
1. **故事类型**：爱情 / 悬疑 / 喜剧 / 古装 / 都市 / 科幻 / 惊悚 / 其他
2. **平台/载体**：短剧（抖音/快手）/ 网络电影 / 院线电影 / 长剧集 / 舞台剧 / 广播剧
3. **核心矛盾或一句话概念**：用户想讲什么故事？有没有初步想法？
4. **主角设定**：性别、职业、性格关键词（可选，如果用户有想法）
5. **大概时长/集数**：短片 / 单集 / 多集连续剧（几集）

### 选问项（根据情况判断是否需要）
- 参考风格或对标作品
- 特殊要求（方言、历史背景、真实事件改编等）
- 目标受众（年龄段、性别）

---

## 第二步：输出剧本结构

根据用户确认的信息，先输出**故事大纲**，结构如下：

```
【剧名】（暂定）
【类型】
【平台】
【一句话故事】
【核心冲突】
【主要人物】
  - 主角：姓名 / 性格 / 核心欲望 / 核心恐惧
  - 对手/配角（视情况）
【故事结构】
  - 开篇钩子（前30秒/第一幕）
  - 第一转折点
  - 中段发展
  - 高潮
  - 结局
```

征得用户确认或修改后，再进入正文写作。

---

## 第三步：剧本正文格式规范

### 中文剧本格式（通用）

```
第X集 / 第X场

场景标题：室内/室外 + 地点 + 时间（日/夜/黄昏）

场景描述（客观描述，不超过3行，交代环境与人物状态）

人物名字
（括号内写动作/表情/语气——可选）
台词内容。

动作行（描述人物动作，用第三人称现在时）

人物名字
台词内容。
```

### 短剧格式（抖音/快手竖屏短剧）

- 每集控制在 **2—5分钟**，约 **400—800字** 剧本文字
- 每集必须有**钩子开头**（悬念/冲突/反转）
- 每集结尾必须有**悬念或情绪爆点**（让观众想看下一集）
- 台词要口语化、节奏快，避免书面语长句

### 院线/长剧格式

- 遵循标准格式：场景行（全大写）、动作行、对白行
- 人物名称在对白上方居中加粗
- 括号内注明必要的表演指示，但不过度

---

## 第四步：写作原则

### 人物
- 每个人物都有**清晰的欲望**（想要什么）和**明确的障碍**（什么阻止他）
- 台词要有**人物专属语气**，避免所有人说话方式一样
- 配角不是道具，给他们1—2个鲜明特征

### 台词
- 少说废话，台词要"做两件事"（推进情节 + 揭示人物）
- 尽量用**动作替代台词**表达情绪
- 避免"话剧腔"，贴近真实口语

### 节奏
- 场景不宜过长，超过3页没有新信息就要剪掉
- 用空白行控制呼吸感
- 高潮前要有充分的铺垫

### 钩子设计（短剧必须）
- 前3秒：视觉冲击或强烈悬念
- 情绪过山车：每隔30—60秒有一个小情绪波动
- 集末反转：出乎意料但合乎情理

---

## 第五步：迭代与修改

完成初稿后，主动问用户：
1. 哪个场景/人物需要调整？
2. 台词风格是否符合预期？
3. 节奏是快了还是慢了？

根据反馈定向修改，不要整体重写，精准改动。

---

## 附：常用剧本术语参考

| 术语 | 含义 |
|------|------|
| INT. / EXT. | 室内 / 室外（英文格式） |
| V.O. | 画外音（人物不在画面内） |
| O.S. | 画外声（人物在场但不在画面里） |
| CONT'D | 人物台词连续（中间有动作描述打断） |
| SMASH CUT | 硬切，强调突然转换 |
| MONTAGE | 蒙太奇段落 |
| 钩子 / Hook | 吸引观众继续看的悬念装置 |
| 第一幕/二幕/三幕 | 经典三幕结构 |
| 内心独白 | 人物内心想法，通常用斜体或括号标注 |

---

## 注意事项

- **不写违法、暴力、色情内容**
- 历史题材注意史实准确性，如有虚构需明确标注
- 真实人物改编需提醒用户注意版权与名誉风险
- 如果用户只给了很少信息，先写一个**微型示范场景（约200字）**让用户感受风格，再展开
